Practical 13: Local Port and Process Auditing
Objective: Map active ports to their specific software processes.

DOS

Advanced TCP Manipulation & Scanning (Task 1)
This practical requires manipulating the TCP header flags to map open ports and evade basic stateless firewalls.
Practical Execution via Nmap:
Open your Kali Linux terminal and execute the following scans against your target IP:
•	1. SYN Scan (Stealth Scan):
o	Command: sudo nmap -sS [Target_IP]
o	Mechanism: Sends a SYN packet. If the port is open, the target replies with SYN/ACK. The attacker immediately tears down the connection with an RST (Reset) packet. It is "stealthy" because the incomplete 3-way handshake is rarely logged by application-level services.
•	2. FIN Scan:
o	Command: sudo nmap -sF [Target_IP]
o	Mechanism: Sends a packet with only the FIN (Finish) flag set. According to TCP RFC 793, closed ports must reply with an RST packet, while open ports must ignore the packet entirely.
•	3. NULL Scan:
o	Command: sudo nmap -sN [Target_IP]
o	Mechanism: Sends a packet with absolutely zero flags set. Similar to the FIN scan, closed ports respond with RST, and open ports drop the packet silently.
•	4. XMAS Scan:
o	Command: sudo nmap -sX [Target_IP]
o	Mechanism: Sets the FIN, PSH, and URG flags simultaneously. This anomalous packet forces closed ports to reply with RST and open ports to ignore it.
Analytical Characteristics: FIN, NULL, and XMAS scans are highly effective against older Unix/Linux systems and stateless firewalls. However, modern Windows operating systems do not strictly follow RFC 793 and will reply with an RST regardless of whether the port is open or closed, making these scans less reliable in a Windows-heavy environment.

Domain 2: Infrastructure Topography (Task 3)
This section maps the physical and logical layout of a network segment using ICMP and TTL (Time-to-Live) manipulation.
Practical Execution:
•	A. Perform Network Scan & Find Active Machines:
o	Command: nmap -sn 192.168.1.0/24 (Replace with your local subnet).
o	Analysis: The -sn flag (formerly -sP) performs a Ping Sweep, disabling port scanning to rapidly identify which host IPs are currently online and responding to ARP or ICMP requests.
•	B. Perform Ping Command:
o	Command: ping -c 4 [Target_IP]
o	Analysis: Tests point-to-point network layer connectivity and measures latency/packet loss using ICMP Echo Requests.
•	C. Find Number of Hops:
o	Command: traceroute [Target_Domain_or_IP] (Use tracert on Windows).
o	Analysis: By incrementally increasing the TTL value of packets, this command forces each router along the path to drop the packet and send an ICMP "Time Exceeded" message back, revealing the exact sequence and number of routing hops.

IV. Viva Preparation: Strategic Defenses
To demonstrate a systems-level understanding during your evaluation, consider how these protocols interact at the operating system level.
Q1: You ran an Xmas Scan and a Null Scan against a target, but every single port showed up as "Closed." However, an Intense Scan showed Port 80 is open. Why did the Xmas/Null scans fail?
•	Answer: Xmas and Null scans rely on a strict interpretation of the TCP RFC 793 to determine port states. While older Unix/Linux systems follow this perfectly, Microsoft Windows operating systems do not. A Windows machine will typically respond with an RST packet to any malformed flag combination, regardless of whether the port is open or closed, rendering these specific stealth scans ineffective against Windows environments.
Q2: During your IP Scanner practical, how does the scanner determine the "Manufacturer" of a device just by scanning the network?
•	Answer: It uses the MAC (Media Access Control) address discovered via the ARP broadcast. The first three bytes (24 bits) of a MAC address are the OUI (Organizationally Unique Identifier). The scanner checks this OUI against a public IEEE database to instantly identify if the hardware belongs to Apple, Cisco, Intel, or another manufacturer.
