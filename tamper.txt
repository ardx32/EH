The objective of this practical is to intercept an HTTP POST or GET request generated by your browser, pause it, modify the underlying data parameters (like a product price or user role), and then forward the manipulated request to the server.

Step 1: Proxy Configuration

Launch Burp Suite and navigate to the Proxy > Intercept tab. Ensure "Intercept is on" is toggled.

Configure your web browser to route its traffic through Burp Suite. This is typically done by setting the browser's HTTP proxy to 127.0.0.1 on port 8080.

(Optional but recommended) Install the Burp Suite CA Certificate in your browser to allow the proxy to intercept HTTPS traffic without throwing security errors.

Step 2: Interception Phase

In your browser, navigate to your vulnerable target application (e.g., an e-commerce lab site like DVWA or bWAPP).

Attempt a transaction. For example, add a "Laptop" priced at $1000 to your cart and click "Checkout."

The browser will appear to freeze. Switch back to Burp Suite.

Step 3: The Tampering Execution

In the Burp Suite Intercept window, you will see the raw HTTP request captured before it reached the network.

Locate the parameter payload. It might look like this in the body of a POST request:
item_id=402&quantity=1&price=1000

Modify the data directly in the text editor. Change price=1000 to price=1.

Click the Forward button to send the tampered packet to the server.

Return to your browser. If the application is vulnerable, your checkout page will now reflect the altered price, demonstrating a successful business logic bypass.
